Mac+laravel安装笔记

发表于 2015-08-19   |   分类于 laravel   |  

Start

不久之前，同事分享过一个主题：史上最牛的PHP框架

由于我入PHP开发时间尚短，了解的框架也仅仅是接触使用的一两个而已。而且都达不到熟悉的程度。

所以对其他的PHP框架竟然是一无所知。于是茫茫然的听完分享。

仅仅是知道了名字叫laravel。

过了一段时间，我无意间搜索PHP框架，看到了一个排名，让我吃惊的是，laravel的全球排名赫然在榜首的位置，霸气凌然。

于是就动了春心，想一观芳容。于是有了后来的安装，挫折，求助，最终安装成功。

这不，还没开始研究具体内容呢，就先来上已发热乎的laravel坎坷的安装之路。

Env

Mac 10.10.3 + Homebrew

Nginx

Nginx是个轻量级的HTTP和反向代理服务器，当然也可以搞其他的诸如IMAP、SMTP等。是俄罗斯的一个大牛写的。

nginx的稳定性和低消耗驰名全球。当年的LAMP基本上都成了LNMP了。
在高连接并发的情况下，nginx是apache不错的替代品。

这里我先把我的nginx配置贴上来，不过仅能作为一点参考，其他的请大家先自行查找。

NOTICE :因为我对nginx的了解也不多，所以这里暂时不多做介绍，回头抽时间研究下nginx的配置，专门来一篇文章入门nginx。 敬请期待

MySQL

mysql的安装，相对来说也简单的多。在Mac上可以直接进行dmg的包安装。省事省心省力。然后下载个图形界面的mysql管理器，一切搞定。

这里说下brew命令安装的过程

具体的启动和其他的内容这里就不做介绍了。brew都会一条条的告诉你的！

PHP56

因为laravel5.1是个LTS(Long Time Support)版本，稳定支持会比较长。laravel5.1对PHP的版本要求是>5.5.9。mac自带了PHP，我的10.10.3的PHP版本是5.5.20，所以能满足laravel的最低要求。所以可以不用再安装php更高版本。

如果有同学的PHP版本不符合要求，那请看下面的brew命令。

注 : 这里的php56安装后，好像会覆盖掉Mac的php版本，没有仔细深究。大家可以刨根问底下。

Composer

composer是个PHP的依赖管理器。一开始我也不知道是个啥东西，后来才知道这个依赖管理器对于PHP开发者来说简直就是个神器啊。她就相当于是终结者中的YellowBook(电话号码记录本)，用来管理PHP的各种依赖。你告诉composer你想找哪个依赖包，composer就能告诉你这个依赖的家庭地址。再也不用无头苍蝇的去google了。

但是让人失望的是，composer大多数的镜像都被无情的封锁了。最近有几个国内镜像站，大家可以去搜一下。

Mac下安装composer很简单啦。非Mac用户本博客暂时不提供支持了，因为一入Mac深似海，此生码农不转Win。say sorry了

注 其实composer就是个可执行的二进制文件，Linux的同学，可以直接下载个composer.phar，然后将~/.bash_profile的PATH更新下，记得source下生效。就可以用啦
